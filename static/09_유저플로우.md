# PinLog 유저플로우

## 1. 가입과 로그인

```mermaid
flowchart TD
    A[소셜 로그인 선택] --> B[Google·Kakao·Naver 인증]
    B --> C{기존 SocialAccount 존재?}
    C -- 예 --> D[로그인 완료]
    C -- 아니오 --> E[User와 Shelf 생성]
    E --> F[필수 약관 안내·동의]
    F --> D
```

## 2. Place 저장과 Record 생성

```mermaid
flowchart TD
    A[카카오맵 Place 검색] --> B[Place 선택]
    B --> C{내 활성 Record 존재?}
    C -- 아니오 --> D[첫 Context 작성]
    D --> E[내부 Place 조회 또는 저장]
    E --> F[Record 생성]
    F --> G[Record 활성화·커밋]
    G -.비동기.-> H[Context 임베딩·Keyword 생성]
    C -- 예 --> J[기존 Record와 Context 확인]
    J --> K[Context 추가·커밋]
    K -.비동기.-> L[새 Context 임베딩·Keyword 생성]
```

AI 처리는 커밋 이후 비동기로 진행하며 Record 활성화를 지연시키지 않습니다. 완료 전에는 Keyword가 비어 있을 수 있습니다.

## 4. Record 수정

```mermaid
flowchart TD

    B --> C[기존 연결과 Record 소프트 삭제]

    G -.비동기.-> H[영향받은 Context의 AI 재처리]
```

## 6. Feed에서 Place 가져오기

```mermaid
flowchart TD
    A[Feed Collection 탐색] --> B[Place 선택]
    B --> C{내 활성 Record 존재?}
    C -- 아니오 --> D[내 Context 작성]
    D --> E[내 Record 생성]
    C -- 예 --> F[기존 Context 확인]
    F --> G[Context 추가 또는 Record 수정]
```

다른 사용자의 Context 원문은 이 흐름에 포함되지 않습니다.
