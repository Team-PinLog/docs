# PinLog 주요 결정사항 변경 이력 — 개정본

## 2026-07-21 정책 통합

### 공식 용어 변경

- Memory 폐기
- 개인 기록 단위: Record
- 독립 서술 단위: Context
- 개인 Archive → Library
- 장소 모음집 → Collection
- 익명 책장 → Shelf
- 발견 피드 → Feed

### 데이터 구조 확정

```text
User + Place = 활성 Record 최대 1개
Record = Context 여러 개
Collection = Record 여러 개
Record = 여러 Collection 포함 가능
```

### Place

- 카카오맵 API 확정
- 검색 결과 조회만으로 Place 저장하지 않음
- Record 생성 시 내부 Place 저장

### 인증

- Google·Kakao·Naver 소셜 로그인
- User와 SocialAccount 분리
- `(provider, provider_user_id)` 유일성

### 빈 데이터

- Context 없는 Record 금지
- Record 없는 Collection 금지
- 마지막 Context 삭제 시 Record 삭제
- 마지막 Record 제거 시 Collection 삭제

### Collection

- 제목만 존재, 설명 없음
- 제목 최대 20자
- 생성 즉시 자동 발행
- 가입 약관에서 자동 발행 고지
- 비공개 전환 없음
- `createdAt ASC` 정렬
- 삭제 시 Record 유지

### Keyword

- AI 프리셋 매핑
- 사용자 읽기만 가능
- 사용자 생성·수정·삭제 불가

### 검색

- Place·Context·Keyword 검색
- Record 단위 반환
- 선택 이유 미표시

### 삭제와 수정

- 사용자 소유 데이터 소프트 삭제
- Record 수정 시 기존 Record 소프트 삭제 후 새 Record 생성
- Collection 연결은 새 Record로 재생성
- 모든 활성 Context 임베딩 재생성
- 회원 탈퇴 시 Shelf와 Follow 관계 제거
- 팔로워 Library에서 탈퇴 Shelf 제거

### 기술 미확정

- GraphRAG
- Java AI 기반 RAG
- 임베딩 처리 방식
